# Software Requirement Specification (SRS)

## 1. Project Title

AI-Powered Phishing Email Detection System

---

## 2. Problem Statement

Phishing emails are one of the most common cyberattacks. Many users fail to identify malicious emails, resulting in credential theft, financial loss, and data breaches.

---

## 3. Objective

To develop a web-based system that analyzes email content and provides a phishing risk assessment.

---

## 4. Scope

The system will:

* Accept email subject
* Accept email body
* Analyze phishing indicators
* Calculate risk score
* Display analysis report

---

## 5. Users

* Students
* Employees
* General internet users

---

## 6. Functional Requirements

FR1: The system shall accept an email subject.

FR2: The system shall accept an email body.

FR3: The system shall identify phishing keywords.

FR4: The system shall calculate a phishing risk score.

FR5: The system shall display the analysis result.

---

## 7. Non-Functional Requirements

NFR1: The system shall provide results within 5 seconds.

NFR2: The system shall have a simple user interface.

NFR3: The system shall be accessible through a web browser.

---

## 8. Technology Stack

Frontend:

* Streamlit

Backend:

* Python

Version Control:

* Git
* GitHub

AI Integration:

* Gemini API (Future Version)

---

## 9. Expected Outcome

The system should help users identify potentially malicious emails and improve cybersecurity awareness.
